AETHER NFC System Developer Guide

ESP8266 + PN532 RFID 기반 실시간 출입통제 및 모니터링 시스템 개발자 매뉴얼

1. 개요 및 시스템 아키텍처

본 프로젝트는 임베디드 디바이스(ESP8266)에서 RFID 카드를 실시간으로 인식하고, 원격 AWS 클라우드 웹 서버에 전송하여 데이터베이스(MySQL) 조회 결과를 바탕으로 출입 승인/거부 여부를 결정하는 클라우드 연동형 스마트 도어록 모니터링 시스템입니다.

+------------------+ HTTP POST +----------------------+ | ESP8266 Device | =============================> | AWS Web Server | | | | (Apache + PHP 8.5) | | - PN532 (I2C) | <============================= | | | - OLED Display | JSON Response | - /api_scan.php | | - Buzzer (D7) | | - /api_register.php| +------------------+ +-----------+----------+ | | SQL Query v +----------------------+ | MySQL/MariaDB | | - Table: users | | - Table: scans | +----------------------+

2. 하드웨어 구성 및 핀 매핑

ESP8266 NodeMCU v2 보드를 메인 컨트롤러로 사용하며, OLED 디스플레이와 PN532 RFID 리더기는 I2C 버스를 공유하여 병렬 연결됩니다.

핀 연결 테이블

기기 핀 (Peripheral) ESP8266 Pin GPIO 비고 (Description)
SSD1306 OLED SDA / PN532 SDA D5 GPIO 14 I2C Data (병렬 공유)
SSD1306 OLED SCL / PN532 SCL D6 GPIO 12 I2C Clock (병렬 공유)
Passive Buzzer + D7 GPIO 13 PWM 오디오 출력 핀
PN532 IRQ D1 GPIO 5 소프트웨어 I2C 더미 핀 (미연결 무방)
PN532 RESET D2 GPIO 4 소프트웨어 I2C 더미 핀 (미연결 무방)
PN532 딥스위치 중요 설정 (I2C 모드)
PN532 모듈을 I2C 통신으로 연결하기 위해 보드 상의 DIP 스위치를 반드시 SEL0: ON (1), SEL1: OFF (0) 상태로 맞춰야 정상 초기화됩니다.

3. 백엔드 API 규격

AWS 클라우드 서버의 데이터베이스 연결 환경과 배포된 REST API 엔드포인트 세부 정보입니다.

데이터베이스 연결 정보

• DB 종류: MariaDB 10.11.14 (Debian 12)
• 접속 계정: rfid_user / 패스워드: rfid_password123
• 데이터베이스명: rfid_system

핵심 API 목록

1. 카드 스캔 검증 및 로깅 (/api_scan.php)

ESP8266 또는 시뮬레이터에서 카드가 태깅되었을 때 호출하는 엔드포인트입니다.

// Request (POST, JSON)
{
  "uid": "4A E1 8B 2F"
}

// Response (JSON)
{
  "success": true,
  "uid": "4A E1 8B 2F",
  "access": true,
  "name": "정재우",
  "role": "개발팀 / 과장"
}

2. 신규 사용자 등록 (/api_register.php)

미등록 카드를 소속, 이름, 전화번호를 매핑하여 DB에 등록합니다.

// Request (POST, JSON)
{
  "uid": "FF 3D E4 78",
  "name": "홍길동",
  "role": "기술팀 / 대리",
  "phone": "010-1234-5678"
}

3. 출입 통계 데이터 반환 (/api_get_stats.php)

필터(filter=today|week|month)에 따라 통계 데이터를 가공하여 반환합니다.

4. CSV 다운로드 내보내기 (/api_export_csv.php)

GET 호출 시 엑셀과 한글이 깨지지 않도록 UTF-8 BOM 처리가 적용된 출입 기록 CSV 파일을 즉시 다운로드 파일 스트림으로 반환합니다.

4. 웹 대시보드 & 통계 분석 페이지

웹 서버 루트 디렉토리(/var/www/html/)에 퍼블리싱된 대시보드와 통계 웹 기능의 상세 내역입니다.

메인 대시보드 (index.html)

• 실시간 모니터링: 3초 간격으로 DB 신규 로그 자동 갱신.
• 자동 팝업 연동: 실물 미등록 카드 감지 시, 카드 UID 필드가 편집 가능한 상태로 신규 등록 팝업 자동 활성화.
• 수동/시뮬레이션 지원: 가상 스캔 시뮬레이터 제공.
• 테마 전환: 다크/라이트 테마 원클릭 스위치 제공 (설정 영구 보존).

통계 분석 (stats.html)

• Chart.js 연동: 시간대별 트래픽 추이(Line), 승인/거부 비중(Doughnut), 부서별 접속량(Bar) 차트 구성.
• 반응형 갱신: 기간 필터(오늘, 7일, 30일) 연동에 따라 차트 데이터가 애니메이션과 함께 동적 로딩.

5. 디바이스 펌웨어 (RfidSystem.ino)

ESP8266에서 돌아가는 통합 펌웨어 소스파일([RfidSystem.ino](file:///c:/Users/SW중심대학사업단/Desktop/AWS/RfidSystem/RfidSystem.ino))의 주요 로직 설명입니다.

동작 로직 상세

• NTP 시계 기능: 와이파이 연결 성공 후, 동기화 서버를 통해 현지 시간(UTC+9)을 상시 유지하고 OLED에 시계 화면을 뿌려줍니다.
• 오디오 피드백 (Buzzer):

  • 부팅음: 도-미-솔 상행 3도 화음 연주
  • 태그 확인음: 단일 고음 비프 한 번("삑-", 80ms)으로 간결하게 처리하여 거부음과 혼동되는 문제를 완전히 수정하였습니다.
  • 승인 성공음: 도-미-솔 상행 빠른 멜로디 재생
  • 거부 경고음: 150Hz 저주파 웅-음 재생

6. 전체 엔드투엔드(End-to-End) 테스트 방법

성공적인 테스트 체크리스트
1. 하드웨어 배선 및 PN532 딥 스위치 상태를 확인합니다.
2. 기기가 켜지면 부팅 멜로디가 나오는지 확인합니다.
3. 와이파이가 정상 연결되면 화면에 디지털 시계가 동기화되어 표기됩니다.
4. 등록된 샘플 카드(예: 4A E1 8B 2F)를 접촉하여 승인 멜로디와 함께 OLED에 본인 정보가 뜨는지 확인합니다.
5. 미등록 카드를 접촉하면 거부 경고음이 울리며, 동시에 대시보드 웹 브라우저 화면에 신규 사용자 등록 팝업이 UID가 채워진 채로 팝업됩니다. 등록을 마치면 이후 동일 카드로 승인 처리가 가능해집니다.
